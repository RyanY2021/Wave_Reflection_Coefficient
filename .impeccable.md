# Impeccable — Project Design Context

This file captures the design context for the Wave Reflection Coefficient
project so future design work stays coherent with how the tool should feel.
Any design skill (craft, layout, polish, etc.) should read this before
writing code.

## Design Context

### Users

Academic researchers and graduate students working with wave-tank experiments.
Their job-to-be-done is computing reflection coefficients from wave-probe
time-series — loading raw `.txt` captures, clipping by travel-time window,
running Goda–Suzuki or Mansard–Funke separation, and inspecting the resulting
`Kr(f)` curve to decide whether an experiment is usable. Many are new to the
codebase and will encounter the Streamlit app without reading the CLI docs
first; options like `bandwidth`, `head-drop`, and the singularity mask need
inline context, not tribal knowledge.

Typical viewing context: a lab workstation or personal laptop, local
Streamlit session, reports previewed in-browser and only saved on download.

### Brand Personality

**Measured. Warm. Exact.**

- **Measured**: the tool is used for quantitative analysis of real experimental
  data. The interface should read as considered and unhurried, never urgent.
- **Warm**: this isn't cold SaaS instrumentation. Cream surfaces and a deep
  warm charcoal — the feel of a well-printed research note, not a dashboard.
- **Exact**: numerics are typeset with care (tabular-lining, monospace for
  technical values), hierarchy is obvious, nothing is decorative-only.

The Streamlit app should feel like a continuation of the HTML report it
produces: same palette family, same type temperament. A user scrolling from
the controls into the embedded preview should not notice a seam.

### Aesthetic Direction

**Editorial lab notebook.** Inherits the existing HTML-report design
language:

- Warm off-white canvas (`--color-background-primary:#ffffff`, secondary
  `#f5f5f2`) in light mode; deep charcoal (`#1a1a1a` / `#2c2c2a`) in dark.
- Serif display stack (`Galaxie Copernicus / Copernicus / Georgia`); mono
  for numeric table cells and precise values.
- Desaturated status pills (olive `#EAF3DE`, amber `#FAEEDA`, brick
  `#FCEBEB`). No saturated/neon status.
- Generous margins, asymmetric composition where it helps, calm rhythm —
  not a grid of identical cards.
- Auto theme via `prefers-color-scheme`, matching how the reports already
  behave, so the embedded preview never mismatches its container.

**Tonal reference**: claude.ai — for its clean, elegant, warm-ivory restraint.
Borrow the sense of careful whitespace and quiet confidence, not specific
UI patterns. A single warm terracotta-ish accent, used very sparingly (a
primary action, a key figure), is in keeping with that reference.

**Anti-references** (hard no):
- AI-SaaS tropes: cyan-on-dark, purple-to-blue gradients, glowing accents,
  glassmorphism, neon CTAs.
- Dashboard pattern: same-size cards in a grid, metric-tile heroes with big
  numbers + gradient accents.
- Side-stripe borders >1px on cards / callouts / alerts.
- Gradient text (background-clip: text with a gradient).
- Monospace-everywhere as shorthand for "technical."
- Fonts from the reflex-reject list (Inter, DM Sans, Space Grotesk,
  Plex family, Fraunces, Playfair, Outfit, Instrument Sans/Serif, etc.).

### Design Principles

1. **Continuity with the report.** App chrome (controls, log strip, path
   pickers) and the embedded report preview should read as one document.
   Same palette, same type temperament, no visible seam where one ends and
   the other begins.

2. **Teach as you go.** The audience includes newcomers. Non-obvious options
   (bandwidth, head/tail drop, method choice, singularity mask) get short
   inline captions or help-affordances. Errors explain what went wrong and
   what to try, not just where it failed. Empty states orient the user.

3. **Quiet warmth.** Cream/charcoal editorial palette, serif display +
   considered body type, tabular numerics. Accents rare and meaningful.
   No neon, no glow, no glassmorphism, no decorative gradients.

4. **Clarity over chrome.** Asymmetric, calm layouts with generous margins.
   Dense-dashboard reflexes (identical card grids, metric tiles, modals
   for every choice) are banned. Hierarchy comes from weight, size, and
   space — not boxes.

5. **Auto theme fidelity.** Honor `prefers-color-scheme` and mirror the
   report's light/dark tokens so the Streamlit preview pane and its
   surrounding app never visually disagree.
